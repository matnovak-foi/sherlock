Stage Three: Examine Results

When detection process is finished, each match is stored into two ways: a tree and a weighted graph for different views of the results.

EXAMINE THE STORED MATCHED

"Examine stored matches" gives the result in a two levels tree. The root is the source directory and the first level is the all the file names of the matched pair and the leaves are the matches of the paired files of their parent node.

Matched Tree




The percentage column shows how long this run of lines is as a percentage of the whole file size: if the two files are exactly the same, this figure should be 100%.

The range of the matches are given by their line numbers

The checkbox on the rightmost can be marked and saved for any suspicious pairs individually or as a group.

To examine the suspicious sections of the submissions,
-Double clicking on the row of the leaves that you want to examine. Then a compare Pane Window will pop up.

EXAMINE THE GRAPH


The data can also be explored using the graph produced by "view matches graph". Each node represents one submission. An edge means there is at least one match between these two submissions The colour of the connecting edges are related to the summed percentage figure over all the matches found for this pair of submissions. Different connected components will be shown in different groups in this Match Graph view.

Viewing the details of the (perhaps multiple) matches
-Move the mouse over an edge or a vertex will show the detail of the matches/vertex in the status bar at the bottom of the window -Left mouse click on an edge will bring up a menu showing all the matches found for this pair, each menu item links to the compare panel, in which the match is shown. Right mouse click will bring up a different menu, which allows the user to mark individual matches for this pair as suspicious or innocent.
